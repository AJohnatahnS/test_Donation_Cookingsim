# กติกา Donation Mode

เอกสารนี้กำหนดกติกาเวอร์ชันแรกสำหรับเชื่อม YouTube และ TikTok เข้ากับ
Cooking Simulator โดยรองรับเฉพาะ Base Career และ Base Kitchen

## 1. ขอบเขต

- ใช้ Runtime Unlock เพื่อเปิดสูตรของ Base Game เฉพาะระหว่างเปิด Mod
- ไม่รองรับ Pizza, Sushi, BBQ หรือ DLC ในเวอร์ชันแรก
- ไม่แก้ไขสถานะปลดล็อกในไฟล์เซฟอย่างถาวร
- หนึ่ง Donation หรือ Gift Combo สร้างสิทธิ์ออเดอร์ได้สูงสุดหนึ่งรายการ
- สูตรที่เสนอให้ผู้ชมต้องสร้างได้จริงในครัวปัจจุบัน

## 2. ระดับ YouTube

YouTube Super Chat และ Super Sticker ใช้ยอดเงินจริงเป็นเกณฑ์
สกุลเงินอื่นต้องแปลงเป็นเงินบาทก่อนจัดระดับ โดยบันทึกทั้งยอดเดิม
สกุลเงินเดิม อัตราแลกเปลี่ยน และยอดเงินบาทที่ใช้ตัดสิน

| ยอดหลังแปลงเป็น THB | ระดับ | การเลือกเมนู | เวลาทำออเดอร์ |
|---:|---|---|---:|
| ต่ำกว่า 20 บาท | Support | ไม่มีออเดอร์ แสดง Overlay เท่านั้น | - |
| 20-49 บาท | Standard | ระบบสุ่ม 1 เมนูง่าย | ไม่จำกัด |
| 50-99 บาท | Viewer Choice | เลือก 1 จาก 3 เมนูง่าย | ไม่จำกัด |
| 100-299 บาท | Priority | เลือก 1 จาก 3 เมนูระดับทั่วไป | 12 นาที |
| 300-499 บาท | Challenge | เลือก 1 จาก 3 เมนูระดับกลางขึ้นไป | 10 นาที |
| 500-999 บาท | Hard Challenge | เลือก 1 จาก 3 เมนูยาก | 8 นาที |
| 1,000 บาทขึ้นไป | Boss Order | เลือก 1 จาก 3 เมนูยากที่สุดที่ทำได้ | 6 นาที |

### YouTube Membership

Membership ไม่ควรคำนวณจากราคาที่คาดเดา เพราะระดับสมาชิกของแต่ละช่องต่างกัน
ให้กำหนด mapping ต่อระดับสมาชิกใน configuration ภายหลัง โดยค่าเริ่มต้นคือ:

| Event | สิทธิ์เริ่มต้น |
|---|---|
| สมาชิกใหม่ | Viewer Choice แบบไม่จำกัดเวลา |
| ต่อสมาชิกหรือ Milestone | Standard แบบไม่จำกัดเวลา |
| Gift Membership 1-4 คน | Priority |
| Gift Membership 5-9 คน | Challenge |
| Gift Membership 10 คนขึ้นไป | Boss Order |

## 3. ระดับ TikTok

TikTok ใช้จำนวน Coins รวมของ Gift Combo เป็นเกณฑ์ ไม่ใช้ชื่อ Gift เป็นเงื่อนไขหลัก
เพราะรายการ Gift ราคา และ availability อาจเปลี่ยนตามประเทศหรือช่วงเวลา

```text
totalCoins = coinValue * repeatCount
```

| Coins รวม | ระดับ | การเลือกเมนู | เวลาทำออเดอร์ |
|---:|---|---|---:|
| 1-19 | Support | ไม่มีออเดอร์ แสดงชื่อ Gift และเอฟเฟกต์เท่านั้น | - |
| 20-99 | Standard | ระบบสุ่ม 1 เมนูง่าย | ไม่จำกัด |
| 100-499 | Viewer Choice | เลือก 1 จาก 3 เมนูง่าย | ไม่จำกัด |
| 500-999 | Priority | เลือก 1 จาก 3 เมนูระดับทั่วไป | 12 นาที |
| 1,000-2,999 | Challenge | เลือก 1 จาก 3 เมนูระดับกลางขึ้นไป | 10 นาที |
| 3,000-9,999 | Hard Challenge | เลือก 1 จาก 3 เมนูยาก | 8 นาที |
| 10,000 Coins ขึ้นไป | Boss Order | เลือก 1 จาก 3 เมนูยากที่สุดที่ทำได้ | 6 นาที |

Gift ที่ส่งต่อเนื่องต้องรอ event จบ Combo ก่อนสร้างสิทธิ์ ห้ามสร้างออเดอร์ใหม่
ทุกครั้งที่ repeat count เพิ่มขึ้น

## 4. การเลือกเมนู

### Standard

- ผู้ชมไม่ต้องเลือก
- ระบบสุ่มหนึ่งเมนูจากกลุ่มเมนูง่าย
- สร้างออเดอร์ทันทีเมื่อถึงคิว

### Viewer Choice ขึ้นไป

1. เมื่อสิทธิ์ขึ้นถึงหัวคิว ระบบสุ่มเมนูที่ไม่ซ้ำกันสามรายการ
2. Overlay แสดงหมายเลข `1`, `2`, `3` พร้อมชื่อเมนู
3. ผู้สนับสนุนมีเวลา 20 วินาทีในการตอบ
4. รับเฉพาะข้อความ `1`, `2`, `3` หรือ `!pick 1`, `!pick 2`, `!pick 3`
5. ตรวจเจ้าของสิทธิ์ด้วย platform user ID ห้ามตรวจด้วย display name
6. เมื่อเลือกสำเร็จ ให้ปิดการรับคำตอบของสิทธิ์นั้นทันที

หากหาเมนูได้ไม่ครบสามรายการ ให้แสดงเท่าที่มี หากไม่มีเมนูที่สร้างได้
ให้พักสิทธิ์ไว้และลองใหม่เมื่อสถานะเกมพร้อม

## 5. Timeout และ Fallback

- หากไม่ตอบภายใน 20 วินาที ระบบสุ่มจากตัวเลือกที่แสดง
- หากคำตอบไม่ถูกต้อง ให้แจ้งเตือนหนึ่งครั้ง แต่ไม่ต่อเวลา
- หากสูตรที่เลือกสร้างไม่ได้ ให้ลองตัวเลือกที่เหลือตามลำดับแบบสุ่ม
- หากทุกตัวเลือกสร้างไม่ได้ ให้คืนสิทธิ์เข้าหัวคิวและสร้างตัวเลือกใหม่หนึ่งครั้ง
- หากลองใหม่แล้วยังไม่ได้ ให้ยกเลิกออเดอร์และบันทึกเป็น `FAILED_GAME_NOT_READY`
- Donation/Gift ที่รับแล้วต้องมีผลลัพธ์สุดท้ายเสมอ: `COMPLETED`, `EXPIRED`,
  `FAILED` หรือ `CANCELLED`

## 6. กติกาเวลา

- Standard และ Viewer Choice ไม่มีเวลาหมดอายุ
- Priority ขึ้นไปเริ่มนับเวลาเมื่อ Mod ยืนยันว่าออเดอร์ถูกสร้างแล้ว
- เวลาเลือกเมนูและเวลารอคิวไม่นับรวมกับเวลาทำอาหาร
- เมื่อเกม Pause ให้หยุดเวลาออเดอร์
- เมื่อเกมกลับเมนูหลัก ให้พักออเดอร์และคิว ไม่ถือว่าผู้เล่นแพ้
- เมื่อหมดเวลา ให้จบเฉพาะออเดอร์นั้นและแสดงผลบน Overlay
- ห้ามใช้ `Time.timeScale = 0` เพื่อทำ timer เพราะจะกระทบทั้งเกมและระบบเชื่อมต่อ

เวลาในตารางเป็นค่าเริ่มต้น ต้องเก็บเวลาทำจริงอย่างน้อย 30 ออเดอร์ก่อนปรับสมดุล

## 7. คิวและข้อจำกัด

- เปิดหน้าต่างเลือกเมนูได้ครั้งละหนึ่งคน
- มี Donation Order อยู่ในเกมพร้อมกันได้สูงสุด 3 รายการ
- คิวรอสูงสุด 20 สิทธิ์
- เมื่อออเดอร์ในเกมเต็ม ให้พักการประมวลผลคิว
- ประมวลผลตามเวลาที่รับ event ก่อนหลัง ไม่แซงคิวตามมูลค่า
- Boss Order มีได้พร้อมกันสูงสุดหนึ่งรายการ
- Event ID เดิมต้องประมวลผลได้ครั้งเดียวเพื่อป้องกันโดเนตซ้ำ

เมื่อคิวเต็ม:

- รับและบันทึก event ไว้
- ไม่สร้างสิทธิ์ gameplay เพิ่ม
- แสดง Overlay ขอบคุณ
- บันทึกผลเป็น `QUEUE_CAP_REACHED`

ต้องประกาศข้อจำกัดคิวบนหน้า Stream ให้ผู้ชมเห็นก่อนสนับสนุน

## 8. การจัดกลุ่มความยากของเมนู

เวอร์ชันแรกแบ่งสูตรเป็นสามกลุ่ม:

| กลุ่ม | ใช้กับระดับ |
|---|---|
| Easy | Standard, Viewer Choice |
| Normal | Priority, Challenge |
| Hard | Hard Challenge, Boss Order |

ห้ามอาศัยชื่อสูตรเพื่อเดาความยาก ให้เริ่มจากรายการกำหนดเองใน configuration
แล้วปรับจากข้อมูลเวลาทำจริง อัตราสำเร็จ และจำนวนขั้นตอนของแต่ละสูตร

ระบบเลือกสูตรต้องตัดรายการต่อไปนี้ออก:

- สูตรนอก Base Game
- สูตรที่ไม่มีอุปกรณ์หรือวัตถุดิบในครัวปัจจุบัน
- สูตรที่มีออเดอร์เดียวกันกำลังทำอยู่ เมื่อยังมีตัวเลือกอื่น
- สูตรที่เพิ่งถูกเลือกใน 5 ออเดอร์ล่าสุด เมื่อยังมีตัวเลือกอื่น
- สูตรที่ถูกปิดใช้งานใน configuration

## 9. Flow ของผู้ชม

```text
ส่ง Super Chat, Super Sticker หรือ TikTok Gift
-> ระบบตรวจ event ซ้ำและคำนวณระดับ
-> สร้างสิทธิ์แล้วใส่คิว
-> เมื่อถึงคิว ระบบสุ่มหรือแสดงสามเมนู
-> ผู้สนับสนุนตอบ 1, 2 หรือ 3
-> ระบบส่งคำสั่งไปยัง Mod
-> Mod สร้างออเดอร์และยืนยันผล
-> เริ่ม timer เมื่อเป็นออเดอร์แบบจำกัดเวลา
-> Overlay แสดงผลสำเร็จ หมดเวลา หรือข้อผิดพลาด
```

## 10. ค่าที่ต้องทำเป็น Configuration

ค่าต่อไปนี้ห้าม hardcode ใน Mod:

- เกณฑ์เงินบาทของ YouTube
- เกณฑ์ Coins ของ TikTok
- mapping ของ Membership
- เวลาเลือกเมนู
- เวลาของแต่ละระดับ
- จำนวน Donation Order สูงสุด
- ขนาดคิวสูงสุด
- รายการสูตรและระดับความยาก
- cooldown และรายการสูตรที่ปิดใช้งาน

## 11. เกณฑ์ยอมรับเวอร์ชันแรก

กติกาถือว่าพร้อมใช้งานเมื่อทดสอบครบ:

- YouTube และ TikTok จัดระดับได้ถูกต้องทุกช่วง
- Gift Combo หนึ่งชุดสร้างสิทธิ์เพียงหนึ่งรายการ
- เฉพาะเจ้าของสิทธิ์เท่านั้นที่เลือกเมนูได้
- Timeout แล้วสุ่มเมนูได้
- Standard และ Viewer Choice ไม่หมดเวลา
- ออเดอร์ราคาแพงเริ่ม timer หลังเกมสร้างออเดอร์สำเร็จ
- คิวไม่เกิน 20 และออเดอร์พร้อมกันไม่เกิน 3
- Event ซ้ำไม่สร้างออเดอร์ซ้ำ
- สูตรที่เลือกมาจาก Base Game และใช้ได้ในครัวปัจจุบัน
- ปิด Mod แล้วสถานะปลดล็อกสูตรในเซฟไม่เปลี่ยน
